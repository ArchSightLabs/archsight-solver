## v1.3.0

发布时间：2026-05-31

本次更新专注于**提升专业工作台的交付品质**，解决 1.2.0 中遗留的交互体验和报告图表痛点，为您带来更顺畅、更可靠的建模与计算体验：

🚀 核心体验升级

- **计算书图表“所见即所得”**：彻底修复 Word 导出图形不一致的问题。现在导出的计算书中，无论是节点、构件、荷载还是计算结果标注，均与网页端完全一致，不再出现图形降级或信息丢失。
- **界面与交互全面统一**：梁系、平面框架、平面桁架的界面均统一为“模板 / 基本 / 对象 / 文本 / 表格”五大明确的页签，降低学习成本，操作更直观。
- **大模型图形自适应增强**：全面提升了超大复杂模型的图形可读性。结构预览将根据节点规模智能适配；在导出超宽的计算书插图时，系统会自动切换横向页面排版，有效避免了图形裁切、过度压缩与图例拥挤。

✨ 功能与模板增强

- **专业模板库全面扩充**：相比 1.2.0 版本，系统大幅丰富了三类结构的内置模板库。不仅囊括了各类常见的经典梁系工况，还补充了诸如“坡屋面门式刚架”、“平行弦桁架”等大量高频专业结构，助您一键生成并直接复用复杂受力模型。
- **更高效的参数编辑**：支持从“对象”页签直接快捷编辑荷载工况与组合；同时将“表格”页签升级为干净的只读摘要看板，让您能极速核对模型所有关键数据。
- **更严谨的防错机制**：针对桁架等特定结构，系统现在会自动拦截类似“弹性支座”等不符合结构假定的异常输入，从源头避免由于误操作导致的错误计算。

🛠 系统架构与工程治理

除了肉眼可见的体验提升，我们在本次版本中也对底层架构进行了深度的重构与治理，确保系统更加健壮：

- **专业术语与数据一致性**：引入了严格的语义“防漂移”机制。现在，无论是在输入面的对象编辑、主控画布，还是输出面的结果预览与计算书，所有的结构元件术语和力学口径都保持了高度的一致性，消除任何可能的工程歧义。
- **前端核心模块重构**：全面收敛了三类结构的基础建模区、文本模型解析以及计算书结构图绘制逻辑。底层架构的统一不仅大幅提升了系统的稳定性，也为后续承载更复杂的力学分析打下了坚实基础。
- **支持私有化部署**：新增 Docker 快速部署脚本与标准操作指南，方便您将计算内核安全、快捷地部署在自有服务器环境中。

## v1.2.0

发布时间：2026-05-29

主要变化：

- 新增公开验证工程入口与验证投稿闭环，覆盖公开案例浏览、投稿包生成、Issue 模板和后端投稿预审。
- 增强工作台工程图可读性，补充梁系、平面桁架、平面框架的尺寸标注、关键结果图和视觉基线。
- 统一计算书图形契约，导出报告优先使用前端同源工程图，并增加导出图形前置校验。
- 扩展公开 benchmark 数据、验证目录摘要、OpenAPI / JSON Schema 契约和 MCP 能力文档。
- 优化大模型编辑、工程树信息密度、公开案例可读性和桁架支座/杆件长度建模表达。
- 修复图表 tooltip HTML 注入风险，补充 HTML 转义、图形契约、投稿包和大模型限制相关测试。

## v1.1.0

发布时间：2026-05-26

主要变化：

- 新增 `/api/jobs` 异步作业接口与 `/api/contracts/schemas` JSON Schema 契约入口。
- 新增 `archsight-solver-tool` CLI 通用入口，扩展 MCP 计算、敏感性分析和 benchmark 工具。
- 新增公开 benchmark catalog、runner、report，并在 CI 中生成验证报告。
- 新增 API Reference、ASMS-JSON 结构力学数据协议、AIOS 集成文档和 benchmark 验证报告。
- 引入 SciPy sparse 求解后端、跨数上限配置、输出精度控制和求解器诊断，增强大跨数与大自由度模型支撑。

## v1.0.0

发布时间：2026-05-25

主要功能：

- 首个公开版本，交付梁系、平面桁架、平面框架三类线弹性静力分析工作台。
- 支持参数建模、结构计算、敏感性分析、结果图表、模板库和 Word / Excel 计算书导出。
- 提供 `/api/calculate`、`/api/sensitivity`、`/api/export` 等基础接口。
- 结果口径覆盖挠度、弯矩、剪力、节点位移、杆件轴力、轴应力和支座反力。
