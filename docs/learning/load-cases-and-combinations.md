# 荷载工况、荷载组合与风荷载

结构计算通常不是只算“一组荷载”。同一个结构模型在不同荷载场景下会有不同响应，因此需要区分荷载工况和荷载组合。

ArchSight Solver 当前把多工况和组合定位为线弹性静力分析中的荷载场景管理能力，不替代规范荷载取值、风荷载体型系数计算、抗震组合或工程签审。

## 什么是荷载工况

荷载工况是一组有明确来源和物理含义的荷载。它回答的是：

```text
这个结构在某一类荷载作用下会怎样响应？
```

常见工况包括：

- 恒载工况：自重、固定设备、永久构造荷载。
- 活载工况：楼面活荷载、检修荷载、临时使用荷载。
- 风荷载工况：某一方向或某一分布口径的风作用。
- 温度工况：整体升温、降温或局部温度作用。
- 施工工况：安装、吊装、临时支撑拆除等阶段性荷载。

在软件中，工况通常有编号和名称，例如：

```text
DL：恒载
LL：活载
WL：风荷载
TL：温度作用
```

## 什么是荷载组合

荷载组合是把多个工况按系数叠加，用于查看组合后的结构响应。它回答的是：

```text
几个工况按指定系数组合后，结构最不利响应是多少？
```

例如一个简化组合可以写成：

```text
1.2DL + 1.4LL
```

或者：

```text
1.0DL + 1.0WL + 1.0TL
```

这些表达只说明“如何叠加已定义的荷载工况”。实际工程中的组合系数、分项系数、可变荷载组合值系数、风荷载参与系数等，应按对应地区规范和工程条件确定。

## 工况和组合的关系

可以把工况理解为“原始荷载场景”，把组合理解为“按系数叠加后的检查场景”。

| 对象 | 作用 | 示例 |
|---|---|---|
| 工况 | 保存一组来源明确的荷载 | `DL`、`LL`、`WL`、`TL` |
| 组合 | 按系数叠加多个工况 | `1.2DL + 1.4LL` |
| 包络 | 从多个工况或组合中取控制结果 | 最大位移、最大弯矩、最大轴力 |

不要把多个模型误认为多个工况。多工况应共享同一套节点、构件、支座和材料，只改变荷载场景。

例如同一个平面框架可以这样组织：

- 节点、构件、支座、材料截面是一套模型。
- `LC1` 可以放基本重力荷载，也可以包含某个温度作用。
- `LC2` 可以放另一组荷载，例如风荷载、不同节点荷载或另一个温度场。
- 结果页切换“主结果 / 工况 1 / 工况 2”时，本质是在看同一结构模型在不同荷载工况下的计算结果。

这就是多工况的核心价值：复杂度增加了，但它把工程上天然存在的多种荷载场景分开管理，避免所有荷载混成一组后失去来源。

## 风荷载在当前产品中的表达

当前开源核心没有内置规范风荷载生成器。它不会根据建筑高度、地面粗糙度、体型系数、风振系数或地区基本风压自动生成设计风荷载。

在当前产品中，风荷载应作为用户已经确定的外部荷载输入，例如：

- 平面框架中的节点水平力。
- 平面框架中的构件分布荷载。
- 平面桁架中的节点水平力或竖向等效节点力。
- 梁系中的等效横向分布荷载或集中荷载。

也就是说，当前 Solver 负责：

```text
已知风荷载 -> 结构响应
```

不负责：

```text
建筑参数 -> 规范设计风荷载
```

## 不同结构类型中的工况价值

| 分析对象 | 多工况价值 | 典型工况 |
|---|---|---|
| 梁系 | 很常见，适合比较恒载、活载、不同布置荷载和组合挠度 / 弯矩 | 均布恒载、跨中集中活载、局部分布荷载 |
| 平面桁架 | 很常见，适合比较节点荷载、自重、风吸力 / 风压力和施工阶段荷载 | 节点竖向荷载、水平风荷载、杆件自重等效荷载 |
| 平面框架 | 最复杂也最需要，适合管理节点荷载、构件荷载、风、温度和组合 | 重力荷载、侧向风荷载、温度作用、组合工况 |

从终态产品看，三类模块都应具备一致的工况和组合体验。当前底层契约和求解链路已支持 `loadCases` / `loadCombinations`，但完整可视化编辑体验优先落在平面框架，梁系和平面桁架仍需要补齐 UI。

## 结果查看时要注意什么

使用多工况时，结果页必须先确认当前查看的是哪一类结果：

- 主结果：通常来自基本荷载或默认组合。
- 工况结果：只来自某个单独工况。
- 组合结果：来自多个工况按系数叠加。
- 包络结果：来自多个结果中的控制值。

同一个构件在不同工况下可能控制指标不同。例如：

- 恒载可能控制梁跨中正弯矩。
- 风荷载可能控制框架顶部水平位移。
- 温度作用可能控制柱脚水平反力或构件轴力。
- 组合结果可能控制最终最大内力。

因此，多工况模型中不能只看一个最大值，还要看它来自哪个工况或组合。

## 与温度荷载的关系

温度荷载适合放入独立工况。例如：

```text
TL+：整体升温
TL-：整体降温
```

然后可以按需要与其他工况组合：

```text
DL + TL+
DL + WL + TL-
```

当前产品只支持二维平面框架构件均匀温度荷载。梁系和桁架的温度作用边界见 [温度荷载与结构类型](temperature-loads.md)。

温度工况单独管理尤其有价值，因为它经常需要单独查看：

- 构件轴力。
- 支座反力。
- 节点位移和变形趋势。
- 与恒载、风荷载组合后的控制结果。

## 当前对外口径

当前版本的荷载场景能力应这样表述：

```text
ASMS-JSON 与后端求解链路支持梁系、二维平面桁架和二维平面框架的荷载工况与荷载组合。当前完整可视化编辑体验优先落在二维平面框架；梁系和二维平面桁架的多工况 UI 属于后续补齐项。
```

风荷载应这样表述：

```text
当前支持把已确定的风作用作为节点荷载或构件荷载输入并参与工况/组合分析；暂不提供规范风荷载自动生成、风振分析或抗风设计结论。
```
