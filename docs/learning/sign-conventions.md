# 符号与方向约定

本文说明 ArchSight Solver 中荷载方向、位移方向、挠度符号和内力符号的公开约定。它是 [ASMS-JSON / Model Schema](../asms-json-schema.md) 中“统一单位与符号约定”的延伸阅读。

符号约定是结构软件里最容易造成误解的部分。同一个物理动作，例如“向下的竖向荷载”，在不同建模对象里可能用不同输入习惯表达。关键不是所有地方都强行同一个正负号，而是每个字段必须有明确、稳定、可复核的方向约定。

## 两套容易混淆的坐标

### 工程坐标

ASMS-JSON 中的结构坐标采用工程直角坐标：

```text
X 向右为正
Y 向上为正
```

因此在平面桁架和平面框架中：

- `fxKn > 0` 表示向右。
- `fxKn < 0` 表示向左。
- `fyKn > 0` 表示向上。
- `fyKn < 0` 表示向下。

### 屏幕坐标

浏览器 SVG / Canvas 的屏幕坐标通常是 Y 向下为正。前端绘图会做坐标映射，让用户看到的图形符合工程直觉。

所以不要用屏幕像素坐标判断工程正负号。工程输入、API、计算书和 benchmark 一律以 ASMS-JSON 的工程坐标为准。

## 为什么梁系和桁架/框架的竖向荷载符号不同

当前项目保留了两种常见习惯：

| 对象 | 字段 | 约定 |
|---|---|---|
| 梁系 | `q`、`pointLoadKn`、梁系 `loads` 中的荷载值 | 正值按当前梁系 UI 习惯表示向下荷载 |
| 平面桁架 | 节点 `fyKn`、杆件等效 `global_y` 荷载 | `fyKn > 0` 向上，向下荷载应写负值 |
| 平面框架 | 节点 `fyKn`、构件 `global_y` 荷载 | `fyKn > 0` 向上，向下荷载应写负值 |

这不是力学矛盾，而是入口习惯不同：

- 梁系常以“竖向向下荷载强度 q”为主输入，很多教学算例也直接写 `q = 10 kN/m` 表示向下均布荷载。
- 桁架和框架是通用节点-杆件/构件坐标模型，更适合遵守全局坐标正方向：Y 向上为正，所以向下为负。

产品和文档必须显式提示这一点，尤其是在“同一个竖向向下荷载”从梁系迁移到桁架或框架时，正负号不能机械照抄。

## 梁系方向约定

梁系主路径面向连续梁、简支梁和悬臂梁，输入习惯偏工程梁图：

- `q = 10` 表示 10 kN/m 向下均布荷载。
- `pointLoadKn = 20` 表示 20 kN 向下集中荷载。
- 线性分布荷载 `qStartKnPerM / qEndKnPerM` 正值也按向下荷载理解。

梁系结果中需要区分两类值：

| 结果 | 读法 |
|---|---|
| `summary.maxDeflectionMm` | 最大挠度控制值，取绝对值，适合服务性校核 |
| `v_data` / 挠度曲线 | 保留曲线符号，用于图形形态和数据曲线 |
| 支座反力、剪力、弯矩曲线 | 保留求解器符号，用于平衡和图形判断 |

实践建议：

- 做校核、排序和摘要时，优先使用 `summary.maxDeflectionMm` 这类绝对控制值。
- 做图形和曲线解释时，保留符号，但要在图例中说明方向约定。
- 不要用“最大挠度为正/负”判断荷载方向；最大挠度摘要本身是控制量，不是方向量。

## 平面桁架方向约定

平面桁架节点自由度为：

```text
ux / uy
```

方向约定：

- `ux > 0`：节点向右位移。
- `uy > 0`：节点向上位移。
- `fxKn > 0`：向右节点力。
- `fyKn > 0`：向上节点力。

因此，节点竖向向下荷载应写成：

```json
{"type": "nodal", "node": "N3", "fxKn": 0, "fyKn": -50}
```

### 桁架杆件荷载等效

理想桁架以节点荷载为主。当前系统也支持可等效为节点荷载的杆件线荷载：

- `direction: "global_x"`：沿全局 X 方向等效。
- `direction: "global_y"`：沿全局 Y 方向等效。
- `selfWeightKnPerM`：按全局 Y 向下处理。

桁架杆件线荷载会先等效为起点和终点节点荷载，再进入求解。它不是把桁架杆件当作受弯梁处理。

### 桁架轴力符号

桁架杆件轴力采用：

- 拉为正。
- 压为负。

`summary.maxAxialForceKn` 是最大轴力绝对值，用来找控制杆件；具体拉压状态应看 `memberResults[].axialForceKn` 或 `forceState`。

## 平面框架方向约定

平面框架节点自由度为：

```text
ux / uy / rz
```

方向约定：

- `ux > 0`：节点向右位移。
- `uy > 0`：节点向上位移。
- `rz > 0`：按右手系约定的绕 Z 轴正转角。
- `fxKn > 0`：向右节点力。
- `fyKn > 0`：向上节点力。
- `mzKnM > 0`：绕 Z 轴正向节点力矩。

### 框架构件荷载方向

框架构件荷载支持两类方向：

| `direction` | 含义 |
|---|---|
| `global_y` | 沿全局 Y 方向作用，正值向上，负值向下 |
| `local_y` | 沿构件局部 y 方向作用，正值沿局部 y 正向，负值反向 |

构件局部坐标按构件起点到终点定义：

- 局部 x 轴：从 `start` 节点指向 `end` 节点。
- 局部 y 轴：垂直于局部 x 轴的正向法线。

对一根从左到右的水平梁，局部 y 正向通常对应全局向上。因此：

```json
{"type": "distributed", "member": "B1", "direction": "local_y", "qStartKnPerM": -18, "qEndKnPerM": -18}
```

表示构件 `B1` 承受向下的局部 y 方向均布荷载。

注意：如果构件起终点反过来，局部坐标也会随之变化。对斜杆或竖向柱，`local_y` 不等于全局竖向。产品界面和计算书应尽量标明荷载是全局方向还是局部方向。

### 框架位移和内力

框架结果中：

- `nodeResults[].uxMm / uyMm / rzRad` 是带方向的节点位移和转角。
- `summary.maxDisplacementMm` 是位移绝对控制值。
- `memberResults` 和 `memberDiagrams` 中的轴力、剪力、弯矩保留符号。
- 结果摘要中的最大构件弯矩通常按绝对值找控制构件。

实践建议：

- 判断“向左还是向右”“向上还是向下”时，看具体节点的 `uxMm / uyMm`。
- 判断“控制值多大”时，看 summary 的最大绝对值。
- 解释构件荷载时必须说明 `global_y` 还是 `local_y`。

## 支座方向约定

### 梁系支座

梁系支座约束：

- `v`：竖向位移。
- `rz` / `θz`：转角。

常见梁系支座：

- `pinned` / `roller`：通常约束 `v`。
- `fixed`：约束 `v` 和 `rz`。
- `free`：释放位移和转角。

### 平面桁架支座

桁架支座只约束平动：

- `pinned`：约束 `ux / uy`。
- `roller`：默认约束 `uy`，释放 `ux`。
- `free`：释放 `ux / uy`。

桁架节点没有 `rz`，因此不应出现转动约束、端部释放或节点弹簧。

### 平面框架支座

框架支座约束：

- `ux`：水平位移。
- `uy`：竖向位移。
- `rz`：转角。

常见框架支座：

- `fixed`：约束 `ux / uy / rz`。
- `pinned`：约束 `ux / uy`，释放 `rz`。
- `roller`：默认约束 `uy`，释放 `ux / rz`。
- 带 `supportAngleDeg` 的 `roller`：约束指定法向位移，释放切向位移。

`supportAngleDeg = 90` 表示竖向法向约束。该字段只用于平面框架；桁架当前不支持滚动支座法向角。

## 图形方向和数据方向

结果图用于帮助理解，但最终方向约定以数据字段为准。

常见注意点：

- 变形图通常会放大显示，不代表真实比例。
- 最大挠度、最大节点位移、最大轴力等 summary 指标通常取绝对控制值。
- 带符号曲线用于看形态、方向和正负区间。
- SVG 屏幕坐标 Y 向下，不能用像素方向替代工程坐标方向。
- 局部坐标图必须结合构件起点和终点理解。

## 常见迁移错误

| 场景 | 错误做法 | 正确做法 |
|---|---|---|
| 桁架节点竖向向下荷载 | 写 `fyKn = 50` | 写 `fyKn = -50` |
| 判断桁架杆件危险性 | 只看 `summary.maxAxialForceKn` | 同时看 `axialForceKn` 的正负和 `forceState` |
| 梁系向下均布荷载迁移到框架梁 | 把 `q = 10` 直接写成 `qStartKnPerM = 10` 的 `global_y` 荷载 | 写成 `qStartKnPerM = -10`、`qEndKnPerM = -10` |
| 框架斜杆局部荷载 | 只写“竖向荷载”但使用 `local_y` | 明确是 `global_y` 还是 `local_y` |
| 判断挠度方向 | 看 `summary.maxDeflectionMm` 的正负 | 看带符号曲线或具体位移字段 |

## 写 UI 文案和计算书时的建议

- 输入框旁边明确写“向下为正”或“Y 向上为正，向下填负值”。
- 梁系文案可以使用“向下荷载 q”。
- 桁架和框架文案应使用“全局 Y 向上为正，竖向向下填负值”。
- 构件荷载必须标注 `global_y` 或 `local_y`。
- 结果摘要使用“最大绝对值”或“控制值”时，不要暗示它有方向。
- 需要方向判断时，引用具体字段：`uxMm`、`uyMm`、`v_data`、`axialForceKn` 等。
