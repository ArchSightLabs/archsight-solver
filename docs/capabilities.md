# 功能与适用边界

本文说明 ArchSight Solver 公开核心的功能范围、适用人群和非目标。工程安全结论、规范设计和施工签审仍需具备资质的专业人员复核。

## 适合谁

- 结构工程师：快速复核常见梁系、二维桁架和二维框架工况。
- 高校教师与学生：用于结构力学教学、作业验证和算例演示。
- 开发者：基于 REST API、CLI、MCP 和 ASMS-JSON 集成确定性求解能力。
- 开源贡献者：补充 benchmark、改善前端交互、增强导出和测试。

## 核心模块

### 梁系

- 连续梁、简支梁、悬臂梁。
- 均布荷载、集中荷载、线性分布荷载和叠加荷载。
- 支座反力、剪力、弯矩、挠度和控制位置。

### 二维平面桁架

- 典型屋架、桥式桁架、悬挑桁架和教学算例。
- 节点荷载和杆件自重等效节点荷载。
- 节点位移、支座反力、杆件轴力和杆件轴应力。

### 二维平面框架

- 门式刚架、显式二维框架和框架梁退化验证。
- 节点荷载、构件分布荷载、构件集中荷载和基础荷载组合。
- 节点位移、支座反力、杆端轴力/剪力/弯矩和构件控制值。

## 工作台能力

- 模板库、基准模板和项目快照。
- 右侧参数面板统一使用“模板 / 基本 / 对象 / 文本 / 表格”五类入口；对象页承载节点、跨段/杆件/构件、支座、荷载和当前对象属性。
- 材料编号用于保留工程语义和导出摘要；线弹性求解仍以各跨段/构件/杆件的 E、A、I 等刚度输入为准。
- 支座编辑按结构体系区分自由度：梁系为 v / θz，平面桁架仅为 ux / uy 平动自由度，平面框架为 ux / uy / rz。
- 结构图、荷载图、内力图、挠度曲线和结果摘要。
- 单因素参数敏感性分析，用于趋势解释和方案比选。
- WORD / XLSX 计算书导出。
- 公开验证工程入口，可直接打开 benchmark 生成的验证模型。

## 计算书与证据链

- DOCX / XLSX 记录材料适用范围、支座体系说明、单位换算、计算方法、平衡校核、公开验证集覆盖和适用边界。
- 平面桁架和平面框架 DOCX 仅使用前端同源结构预览图和模型叠加工程图；缺失时跳过对应插图并写入说明，不插入后端简化兜底图。
- DOCX 图形导出链路提供 Chromium / Firefox / WebKit 三浏览器矩阵入口：`npm --prefix frontend run test:visual:export-docx`。

## 集成能力

- REST API：同步求解、异步作业、预览、敏感性分析、导出和公开案例。
- ASMS-JSON：梁系、二维平面桁架和二维平面框架的统一结构模型协议。
- CLI：本地自动化、批处理和 CI 复核。
- MCP：Agent Host 发现工具、读取 schema、执行公开验证集算例。
- Benchmark：公开验证集、误差阈值和回归报告。

### 异步作业边界

- `/api/jobs` 是本地轻量异步求解入口，用于本机批量计算、Agent 自动调用和避免同步请求阻塞。
- 状态与结果默认写入本地 SQLite，可通过 `ARCHSIGHT_SOLVER_JOB_DB_PATH` 覆盖；执行仍由当前服务进程的 `ThreadPoolExecutor` 承担，并记录执行进程 ID。
- 进程重启或线程句柄丢失后，旧的 `queued` / `running` 记录会在查询时标记为 `COMMON_ASYNC_JOB_ORPHANED` 失败，避免永久挂起；调用方应重新提交作业。
- 该入口不承诺生产级多实例队列、幂等重试、跨主机调度或高吞吐任务编排；生产化部署应接入共享数据库、Redis 或专用任务队列。

## 身份与访问边界

当前开源核心默认直接进入本地工作台，不内置登录、组织、权限或会话管理。

如果后续需要统一身份或多用户能力，应由外部网关、宿主平台或独立集成服务提供身份、权限和审计上下文；Solver 本身只消费这些结果，不在本仓库内实现认证平台。

## 当前明确非目标

- 三维杆系、Frame3D 或空间结构求解。
- 动力分析、稳定分析、非线性材料、几何非线性和接触问题。
- 规范设计、施工安全专项业务和工程签审。
- OpenSeesPy 或其他外部求解器作为运行时内核。
- 云协作、多用户平台、组织权限系统和插件化生态。

这些方向不进入当前开源核心。新增能力应优先服务梁系、二维平面桁架和二维平面框架的可信计算、结果表达、导出质量和 benchmark 覆盖。
