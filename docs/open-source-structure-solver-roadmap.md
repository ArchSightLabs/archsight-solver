# 开源结构力学求解器路线图与定位计划

> 说明：本文件用于路线规划与阶段范围控制，产品需求与验收以规格与测试为准。
> 状态注记：仓库已补齐最小开源硬化闭环（`LICENSE`、`CONTRIBUTING.md`、PR/Issue 模板、基准算例库初版）；下文保留为后续扩展路线。

## Summary
- 结论先说：**合适，而且值得做**，但前提是把项目定位成“**Web 原生、透明可验证的梁系 + 二维平面框架 + 二维平面桁架工作台**”，而不是再去做一个“大而全的工程套件”。
- 借鉴方向应当分层：
  - 借鉴成熟工程软件的，是**模块化产品思维、交付闭环、报告/方案输出能力、场景包装**。
  - 借鉴经典教学型求解器的，是**经典结构力学问题覆盖、教学/工程双场景、低门槛建模、求解过程清晰**。
- 你的项目最适合的切入点是：**把梁系、二维框架、二维桁架作为三个核心模块同步做稳**，再围绕便捷入口、WORD 计算书、可视化与回归门禁持续打磨。

## Positioning Learnings
- 成熟工程软件的优势通常不是单点计算，而是“**工程场景 + 规则校核 + 计算书/方案摘要/风险提示**”的一体化交付。
- 经典教学型结构力学工具的价值在于：把繁琐计算交给软件，把精力留给分析、建模、复核与理解。
- 这两类工具合起来说明一件事：**行业用户真正需要的，不只是“算得出”，而是“算得准、看得懂、录得快、能复用、能交付”**。
- 本仓库是独立开源实现，不包含任何第三方商业软件源码、内部资料、专有规则库或机构背书。
- 本仓库采用 Apache-2.0 许可证，当前公开范围聚焦梁系、二维平面框架和二维平面桁架。

## 最小开源范围
> 目标：先把一个“别人能直接用、能验证、能贡献”的核心做出来，再扩展。

### Must Have
- **计算核**
  - 梁系、二维平面框架、二维平面桁架三类核心求解
  - 连续梁、简支梁、悬臂梁
  - 门式刚架、显式二维框架、典型二维桁架
  - 均布荷载、集中荷载、线性分布荷载与常见节点荷载
  - 支座反力、剪力、弯矩、挠度、最大值位置

  - 每个模块的便捷入口、预置模型与默认参数
- **结果表达**
  - 结构图、荷载图、内力图、挠度曲线
  - 结果摘要、校核结论、关键参数解释
  - 导出 WORD 计算书
  - WORD 中包含输入参数、图形、验算过程与验证结果
- **工程可验证性**
  - 典型算例库
  - 自动化回归测试
  - 参考解对照与误差阈值
- **开源可用性**
  - 清晰 README
  - 安装/运行/开发文档
  - 贡献指南
  - 最小示例工程

### Should Have
- 模板库/工况快照
- 历史记录与恢复
- 敏感性分析
- API 接口文档
- 示例数据集与基准测试集
- 更友好的模块切换、建模引导与结果页交互

## 三段式路线图

### 第一段：当前主线

- 模板库、导出能力、基准算例与错误契约已经进入日常工作流
- 产品价值聚焦在“可信计算 + 快速录参 + 结果可交付”

### 第二段：未来 3 个月重点
- 为三个核心模块补齐高频工况的便捷入口、预置模型和默认参数
- 把 WORD 计算书打磨成可直接复核、可直接交付的输出
- 重做关键交互路径，降低首次建模、参数编辑与结果阅读门槛
- 扩大三大模块的基准算例库，收紧数值阈值与错误契约
- 把文档、安装、贡献和 CI 变成可执行门禁

### 第三段：明确非目标

- 钢筋混凝土/钢结构规范设计全家桶

- 多用户协作、云同步、内置权限系统
- 施工安全模块大拼盘

## 三大核心模块协同推进
> 目标：在不破坏可验证性的前提下，把梁系、二维平面框架、二维平面桁架三条主线共同做成稳定、易用、可交付的工程工具。

### 推进原则
- **三个模块并行，不再追加新分析域**：近期资源只投向梁系、二维框架、二维桁架，不再新增第四类分析模块。
- **先线弹性，后高级分析**：首阶段只做线弹性静力分析，保证结果清晰、边界明确、测试稳定。
- **先自研主核，不接外部引擎**：OpenSeesPy 不作为本项目目标。
- **先高频工况，后泛化能力**：优先覆盖门架、常见框架、典型屋架/支撑桁架等高频对象。
- **先可用性，后花哨能力**：优先做好便捷入口、默认参数、结果页交互和 WORD 计算书。

### 梁系
**定位**
- 作为最基础、最常用、最能体现可信度的核心模块，承担连续梁与常见梁系快速复核任务。

**近期重点**
- 增加高频工况的便捷入口和默认参数
- 强化结果页对控制截面、最大挠度和关键图形的解释
- 强化 WORD 计算书中的验算过程、图形和结果摘要

### 二维平面框架
**定位**
- 作为三个核心模块之一，承接常见门式刚架和显式二维框架分析需求。

**当前状态**
- 已将二维框架求解、前端工作台、导出和基准算例纳入后端测试与公开路线图约束。
- 已完成现状基线、错误体系、导出增强与前端工作台收口验证。
- 下一步重点是门式刚架等高频工况的快捷入口、默认参数和结果页交互，不再继续扩大分析类型。

**建议优先能力**
- 节点坐标、构件连接与支座条件建模
- 节点荷载、构件分布荷载、局部荷载输入
- 位移、转角、支座反力、杆端内力、构件控制值输出
- 结果图联动展示：变形图、内力图、反力图、构件结果表
- 典型框架算例库与回归测试

**验收重点**
- 结果能与教材算例、人工校核或土木朋友确认的参考值对齐
- 常见约束与荷载组合下不会出现“能算但不可信”的情况
- 输入错误时能给出明确、工程化的报错

### 二维平面桁架
**定位**
- 作为三个核心模块之一，面向典型屋架、支撑体系和教学算例的二维分析需求。

**当前状态**
- 已完成二维桁架 v3，并把二维平面桁架的显式建模、求解、预览、导出与回归契约落到后端实现。
- 下一步重点是前端模式接入、典型桁架的快捷入口、桁架图形展示和更完整的基准算例库。

**建议优先能力**
- 节点铰接桁架模型
- 轴力、位移、支座反力输出
- 常见节点荷载、杆件自重/等效荷载输入
- 结构稳定性提示与超长细比/约束缺失的基础检查
- 与框架共享同一套项目、算例、导出与验证体系

**验收重点**
- 轴力分布、节点位移和支反力结果可复核
- 能覆盖市政领域常见的二维桁架示例
- 与框架模型共用数据结构时，不引入混乱的分支逻辑

### 当前不纳入路线的能力
- OpenSeesPy 或其他外部分析内核
- 将 Frame3D / 三维杆系纳入当前开源核心
- 非线性材料与几何非线性
- 动力时程、稳定后分析、复杂约束与接触问题
- 插件化生态、云协作、多用户系统
- Solver 内置登录系统；未来统一身份能力按独立集成议题处理
- 规范设计与施工安全业务

这些能力不是本项目当前目标，不进入近中期路线，也不作为对外承诺。

## 设计原则
- **先收敛再打磨**：先把三类核心模块做成稳定产品，不再继续开新分析域。
- **先核后壳**：先保证结果可信，再追求界面华丽。
- **先交付再扩张**：先把 WORD 计算书、默认参数、便捷入口和 UI 做到可用。
- **先开源核心可用，不做平台化**：先能安装、能跑、能验证，不做插件、云协作和多用户系统。
- **先通用核心，不做行业碎片化适配**：不要一开始就陷入地区规范和施工安全全家桶。
- **先边界清晰，再扩展能力**：当前仓库先保持三类二维静力分析透明可验证。

## 成功标准
- 外部用户可以在较短时间内完成安装并跑通示例。
- 典型算例结果可与参考值稳定对齐。
- 用户能把它当成稳定的工程工具。
- 贡献者可以无障碍新增算例或扩展模块。
- 项目能形成清晰的开源叙事：**不是小号商业软件，而是聚焦三类典型结构的现代结构力学求解器**。
