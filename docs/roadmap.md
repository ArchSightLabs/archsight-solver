# ArchSight Solver 开源路线图

> 本文件用于说明 ArchSight Solver 的公开核心范围、近期建设重点和明确非目标。当前功能边界以 [功能与适用边界](capabilities.md) 为准；接口字段以 [API 参考](api-reference.md) 和 [ASMS-JSON 规范](asms-json-schema.md) 为准；数值可信度以公开验证集和自动化测试为准。

## 产品定位

ArchSight Solver 是面向结构工程师、高校教师和进阶学习者的开源结构力学求解器工作台。当前主线不是“大而全”的工程设计平台，而是把三类典型二维结构分析做成可安装、可验证、可复核、可导出的专业工具：

- 梁系：连续梁、简支梁、悬臂梁。
- 二维平面框架：门式刚架、显式节点-构件模型和框架梁退化验证。
- 二维平面桁架：典型屋架、桥式桁架、悬挑桁架和教学算例。

核心价值是：结果可信、建模快捷、图表清晰、计算书可交付、算例可回归。

## 当前基线

当前公开核心已经形成三类分析对象的最小闭环：

- Web 工作台：梁系、二维平面框架、二维平面桁架的参数建模、结构预览、求解和结果查看。
- 计算结果：支座反力、剪力、弯矩、挠度、节点位移、杆件轴力、杆件轴应力等专业指标。
- 工程输出：结构图、荷载图、内力图、挠度曲线、结果摘要、WORD / XLSX 计算书。
- 数据入口：ASMS-JSON、REST API、CLI、MCP tools、文本模型和公开案例接口。
- 质量门禁：公开 benchmark、错误契约、后端测试、前端单元测试、DOCX 导出三浏览器矩阵和视觉回归说明。

截至 2026-05-31，公开验证集包含 33 个通过算例：

| 分组 | 数量 | 覆盖重点 |
|---|---:|---|
| 梁系 | 12 | 简支梁、悬臂梁、连续梁、均布荷载、集中荷载 |
| 二维平面框架 | 13 | 门式刚架、构件荷载、弹性支座、框架梁退化验证 |
| 二维平面桁架 | 8 | Pratt、Warren、Howe、悬挑桁架、杆件自重 |

验证报告见 [公开验证集报告](verification/benchmark-validation-report.md)。

## 路线原则

- 先巩固三类核心模块，不新增第四类分析域。
- 先保证线弹性静力分析的数值可信度，再扩展高级分析。
- 先统一模型、结果、导出和测试契约，再优化交互细节。
- 先收紧材料、支座、节点、跨段/构件/杆件等专业对象语义，再增加更复杂的建模入口。
- 先服务高频工况和可复核算例，再追求泛化建模能力。
- 先保持开源核心轻量可维护，不在 Solver 内置平台化能力。

当前建设重点已经从“大组件拆分”转为专业建模控制、计算书可信度和契约漂移治理。公开路线只承诺能由代码、导出证据链和自动化测试证明的能力。

## 近期路线

### 未来 1 个月：专业建模控制与入口补齐

目标是让首次使用者能快速进入三类典型结构的标准工况，并能看懂当前模型口径。

- 梁系、二维平面框架、二维平面桁架分别补齐 3 到 5 个高频模板。
- 统一模板命名、默认参数、单位口径和结果摘要字段。
- 梳理工作台右侧参数面板，保持“模板 / 基本 / 对象 / 文本 / 表格”一致入口。
- 统一材料编号与 E/A/I 刚度输入、支座自由度、节点、跨段/构件/杆件在 UI、文本模型、图形和导出中的表达。
- 将公开案例、模板库和计算书导出串成稳定演示路径。

验收口径：

- 每个核心模块至少有 3 个可直接计算的高频模板。
- 模板导入后可完成求解、查看图形结果并导出计算书。
- 三个模块的参数面板一级入口一致，专业术语无明显漂移。

### 未来 2 个月：计算书与结果表达打磨

目标是让计算结果不仅能在页面查看，也能形成可复核的工程交付材料。

- 统一三类模块的计算书结构：输入参数、模型图、荷载信息、关键结果、验证说明和适用边界。
- 补强梁系控制截面、最大挠度位置、支座反力和弯矩/剪力图说明。
- 补强框架节点位移、杆端内力、构件控制值和荷载工况表达。
- 补强桁架节点位移、杆件轴力、轴应力和受拉/受压状态表达。
- 将敏感性分析作为计算书附录能力，不替代基础求解和工程校核。

验收口径：

- WORD / XLSX 导出字段与页面结果、API 结果保持一致。
- 平面框架和平面桁架 DOCX 不展示非 UI 同源的降级结构图，缺失图形时应明确提示并跳过。
- 框架/桁架 DOCX 导出图形链路按 Chromium / Firefox / WebKit 顺序矩阵回归，图形 key 来自共享目录。
- 计算书明确记录单位、符号约定、验证来源和适用边界。
- 桁架计算书不得引入弯矩作为主指标；框架和梁系指标口径各自正确。

### 未来 3 个月：回归门禁与交互收口

目标是让新增能力可验证、可回归，避免路线扩大后牺牲专业可信度。

- 扩大 benchmark 覆盖，增加高频模板对应的公开验证算例。
- 收紧错误契约，保证输入错误能返回工程化、可定位的诊断信息。
- 固化后端测试、前端单元测试、视觉回归和文档一致性检查。
- 优化模块切换、模型编辑、结果总览和导出设置的操作路径。

验收口径：

- 新增模板必须有可运行算例或自动化测试覆盖。
- benchmark 报告可稳定生成，失败时能定位到结构类型、算例和指标。
- 前端关键工作流覆盖梁系、二维平面框架、二维平面桁架三类对象。

## 模块重点

| 模块 | 当前重点 | 不应偏离的边界 |
|---|---|---|
| 梁系 | 高频梁型模板、控制截面说明、挠度/弯矩/剪力图和计算书表达 | 不扩展到复杂桥梁专项、影响线或动力分析 |
| 二维平面框架 | 门式刚架、显式节点-构件建模、节点位移、杆端内力和构件控制值 | 不引入三维杆系、规范承载力设计或施工安全专项 |
| 二维平面桁架 | 典型屋架/桥式桁架、节点位移、杆件轴力、轴应力和稳定性诊断 | 不把桁架杆件当作受弯构件，不输出弯矩主指标 |
| 公共能力 | ASMS-JSON、模板库、公开案例、导出、benchmark、错误契约 | 不让前端、API、CLI、MCP 各自发明一套模型字段 |

## 明确非目标

以下方向不进入当前开源核心，也不作为近中期对外承诺：

- 三维杆系、Frame3D 或空间结构求解。
- 影响线、动力分析、稳定分析、非线性材料、几何非线性和接触问题。
- 规范设计、构件承载力设计、施工安全专项业务和工程签审。
- OpenSeesPy 或其他外部求解器作为运行时内核。
- 云协作、多用户平台、组织权限系统和插件化生态。
- 地区规范碎片化适配和商业软件式模块大拼盘。

如果后续需要上述能力，应作为独立项目或独立扩展路线重新立项，不反向牵引当前三类核心模块扩大范围。

## 成功标准

- 外部用户可以按 README 和快速开始文档完成安装、启动、求解和导出。
- 三类核心模块的典型算例可与参考解、教材解或独立刚度法基线稳定对齐。
- 结构工程用户能清楚看到输入、假定、结果、单位、符号约定和适用边界。
- 贡献者可以通过 ASMS-JSON、benchmark 和测试补充新算例。
- 项目对外叙事保持清晰：这是聚焦梁系、二维平面框架和二维平面桁架的现代结构力学求解器，不是小号商业设计平台。

## 维护规则

- 新增路线项必须说明对应结构类型、响应指标、验证方式和非目标影响。
- 新增模板、导出字段或 API 字段必须同步检查 README、能力边界、API 参考和测试。
- v1.2 后的中英双语与全球开源定位升级按 [中英双语与全球开源定位方案](global-open-source-i18n-plan.md) 分阶段推进，不混入当前发布收尾。
- 新增功能若不能进入 benchmark、测试或可复核样例，应先作为实验能力处理，不写入主路线承诺。
- 路线图只保留阶段方向和验收口径；具体任务拆分应放在 issue、PRD 或任务清单中。
