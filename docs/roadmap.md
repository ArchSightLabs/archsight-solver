# ArchSight Solver 开源路线图

> 本文件用于说明 ArchSight Solver 的公开核心范围、近期建设重点和明确非目标。当前功能边界以 [功能与适用边界](capabilities.md) 为准；接口字段以 [API 参考](api-reference.md) 和 [ASMS-JSON 规范](asms-json-schema.md) 为准；数值可信度以公开验证集和自动化测试为准。

## 产品定位

ArchSight Solver 是面向结构工程师、高校教师和进阶学习者的开源结构力学求解器工作台。当前主线不是“大而全”的工程设计平台，而是把三类典型二维结构分析做成可安装、可验证、可复核、可导出的专业工具：

- 梁系：连续梁、简支梁、悬臂梁。
- 二维平面桁架：典型屋架、桥式桁架、悬挑桁架和教学算例。
- 二维平面框架：门式刚架、显式节点-构件模型和框架梁退化验证。

核心价值是：结果可信、建模快捷、图表清晰、计算书可交付、算例可回归。

## 当前基线

当前公开核心已经形成三类分析对象的最小闭环：

- Web 工作台：梁系、二维平面桁架、二维平面框架的参数建模、模型预览、求解和结果页受力变形查看。
- 计算结果：支座反力、剪力、弯矩、挠度、节点位移、杆件轴力、杆件轴应力等专业指标。
- 工程输出：结构图、荷载图、内力图、挠度曲线、结果摘要、WORD / XLSX 计算书。
- 数据入口：ASMS-JSON、REST API、CLI、MCP tools、文本模型和公开案例接口。
- 质量门禁：公开 benchmark、错误契约、后端测试、前端单元测试、DOCX 导出三浏览器矩阵和视觉回归说明。

截至 2026-06-05，公开验证集包含 61 个通过算例：

| 分组 | 数量 | 覆盖重点 |
|---|---:|---|
| 梁系 | 22 | 简支梁、悬臂梁、连续梁、均布荷载、集中荷载、局部荷载和弹性支座 |
| 二维平面桁架 | 16 | Pratt、Warren、Howe、悬挑桁架、平行弦桁架、杆件自重和专项验证 |
| 二维平面框架 | 23 | 门式刚架、构件荷载、弹性支座、构件集中荷载、模板回归、商业软件对标和框架梁退化验证 |

验证报告见 [公开验证集报告](verification/benchmark-validation-report.md)。

## 路线原则

- 先巩固三类核心模块，不新增第四类分析域。
- 先保证线弹性静力分析的数值可信度，再扩展高级分析。
- 先统一模型、结果、导出和测试契约，再优化交互细节。
- 先收紧材料、支座、节点、跨段/构件/杆件等专业对象语义，再增加更复杂的建模入口。
- 先服务高频工况和可复核算例，再追求泛化建模能力。
- 先保持开源核心轻量可维护，不在 Solver 内置平台化能力。

当前建设重点已经从“大组件拆分”转为专业建模控制、计算书可信度和契约漂移治理。公开路线只承诺能由代码、导出证据链和自动化测试证明的能力。

## 近期路线

### v1.4.0 发布记录与边界

v1.4.0 已于 2026-06-08 发布，主题是“专业建模效率 + 框架荷载场景增强”。本版不新增第四类分析对象，重点收口工作台显式建模效率、平面框架温度 / 支座位移等荷载场景、公开 benchmark 验证证据链和发布前自动化验收。

发布范围：

- `CHANGELOG.md` 和前端 release notes 是 v1.4.0 对外发布说明的事实源；本节只保留路线边界和验收口径。
- v1.4.0 不新增第四类分析对象，仍聚焦梁系、二维平面框架和二维平面桁架。
- 多工况 / 荷载组合在数据契约和求解 payload 中继续收敛；当前完整可视化编辑和结果切换体验优先落在平面框架。
- 温度作用当前只承诺二维平面框架线弹性静力下的构件均匀温差，用于整体升降温引起的伸缩、约束反力、杆端内力和节点位移分析。
- 复制、镜像、阵列属于高效构造工具，发布后仍应按“可撤销、可复核、不过度打扰默认建模流程”的原则继续收敛。

v1.4.0 已完成的发布验收：

- [x] 完成梁系、二维平面框架、二维平面桁架各至少一个内置模板的 UI 冒烟：打开模板、编辑对象、运行计算、查看受力变形和工程图。
- [x] 对平面框架多工况做发布验收：新增两个工况、分别施加普通荷载和温度荷载、切换结果页工况并确认结果图与当前工况一致。
- [x] 对构件均匀温度荷载做发布验收：确认温差、线膨胀系数、材料默认值、支座反力、杆端内力和受力变形图均能正常显示。
- [x] 对撤销 / 重做做发布验收：覆盖新增节点、连接构件、修改荷载、表格批量编辑和构造工具操作。
- [x] 对复制 / 镜像 / 阵列做发布验收：确认不会生成重复编号、悬空构件或不可求解模型，误操作可通过撤销恢复。
- [x] 对网格吸附做发布验收：对象页和表格页的开启状态、步距输入和坐标编辑行为一致。
- [x] 更新 `CHANGELOG.md`、版本号和 release notes 同步脚本输出。

已补充的自动化验收口径：

### v1.5.0：高可用建模工作台大版本

v1.5.0 跳过 v1.4.1，但不再按“荷载场景收口版”发布。本版必须成为面向正式发布的大版本：在不新增第四类分析对象、不放松数值可信边界的前提下，把梁系、平面框架和平面桁架做成更容易上手、更快首算、更有功能感的专业工作台。

本版主线：

- 操作习惯：缩短“新建 / 选模板 / 建模 / 计算 / 导出”的路径，支持从模板页直接生成并计算，内置模板也能打开并首算，减少用户在首次求解前必须理解的对象层级。
- 功能补充：保留荷载工况、荷载组合、结果来源和导出来源一致性作为基础能力，同时补充用户可直接感知的快速建模、工程预设和生成摘要。
- 建模简化：在既有“模板 / 基本 / 对象 / 文本 / 表格”五页签内增加快速生成入口，不新增模块专属一级页签；梁系继续保持“跨段 / 支座 / 荷载”心智，框架和桁架继续保持显式节点-构件/杆件模型的专业边界。

本版优先任务：

- 梁系模板页新增连续梁快速生成、工程预设、生成摘要和生成并计算：按跨数、单跨长度和全跨均布荷载生成跨段、支座与荷载，保留梁系行业常用建模心智。
- 平面框架模板页新增规则框架快速生成、工程预设、生成摘要和生成并计算：按跨数、层数、跨度、层高、梁面均布荷载和顶层水平荷载生成节点、构件、支座和荷载。
- 平面桁架模板页新增平行弦桁架快速生成、工程预设、生成摘要和生成并计算：按节间数、节间长度、桁高和上弦节点荷载生成节点、杆件、支座和节点荷载，不引入桁架假定外的弯矩主指标。
- 三类内置模板新增“打开并计算”：用户可以直接从模板页载入公开验证映射模板并进入首算结果。
- 新建分析对象时增加建模路径选择：快速生成、对象编辑、文本导入分别进入模板、对象、文本页，减少创建后再次寻找入口的摩擦。
- 梁系和平面桁架补齐荷载工况 / 荷载组合 UI，覆盖结果来源切换和导出来源记录；平面框架保持已有多工况能力。
- 结果页在三类模块中统一展示“主结果 / 工况 / 组合”，避免用户误把不同工况结果混看。
- 计算书和 XLSX 导出记录当前结果来源：基本荷载、指定工况或指定组合。
- 结构模型诊断中心在三类模块求解前给出统一诊断：支座约束不足、孤立节点/杆件、荷载未归属工况、组合引用丢失、单位/刚度异常、桁架不适用字段和结果异常放大提示。
- 计算书审阅模式写入“草稿 / 可审阅”状态，并在 DOCX / XLSX 中明确模型假定、单位、符号约定、工况/组合来源、平衡校核、诊断警告和 benchmark 参考。
- ASMS-JSON 契约治理纳入 v1.5.0：三类求解 payload 声明 `schemaVersion`，项目文件记录 ASMS-JSON 契约版本，文本模型 / JSON / 项目导入迁移提供一致性诊断。
- v1.4.0 后的 esbuild 二进制完整性告警修复直接纳入 v1.5.0，不单独发布 v1.4.1。

验收口径：

- 三类模块均能从“模板”页通过快速生成得到可计算模型，并能通过“生成并计算”直接进入首算结果；快速生成应提供工程预设和生成摘要，生成后仍能继续进入对象页精细编辑。
- 三类内置模板均能从模板页通过“打开并计算”完成模板载入和首算，不要求用户手动切到结构计算页再运行。
- 新建分析对象时选择不同建模路径后，创建完成应直接打开对应页签，不要求用户再手动切换右侧参数入口。
- 快速生成不会覆盖五页签信息架构，不会把梁系降级成节点 / 杆件心智，也不会让桁架显示弯矩、剪力作为主校核指标。
- 三类模块均可在 UI 中新增、编辑、删除荷载工况和荷载组合。
- 工况和组合切换后的结构图、受力变形、数据曲线和结果摘要一致。
- API、文本模型、项目文件和计算书中的工况 ID、组合系数和标签无漂移。
- 求解前诊断面板能在三类模块中解释“为什么不能算”或“为什么结果可疑”，并且不会静默允许明显不可解模型进入正常结果口径。
- DOCX / XLSX 计算书能区分草稿与可审阅，并写入 ASMS-JSON 契约版本、结果来源、诊断警告和公开 benchmark 参考。
- `shared/asms-contract-fields.json`、JSON Schema、OpenAPI、前端 payload builder、项目文件导入和公开文档对 `schemaVersion` 的定义无漂移。
- 发布前按 [v1.5.0 发布验收清单](verification/release-1-5-acceptance.md) 复核快速建模、来源切换和导出来源记录。

### v1.5.0：温度作用边界评估

温度作用不应一次性扩成热分析平台。v1.5.0 只把已实现的平面框架均匀温差作为受支持能力继续记录边界；是否扩展到桁架、梁系或截面梯度，应按结构模型假定分别评估。评估材料见 [温度作用后续评估](temperature-action-evaluation.md)。

可评估方向：

- 平面桁架杆件均匀温差：与二力杆轴向伸缩假定相容，可作为后续增强项。
- 平面框架截面温度梯度：工程价值高，但需要新增截面温度分布、热曲率换算、符号约定和 benchmark。
- 梁系温度梯度：比梁系均匀温差更有工程意义，但应单独定义输入契约和验证基准。

暂不进入当前承诺范围：

- 理想桁架杆件截面温度梯度。
- 瞬态热传导、热-结构耦合。
- 混凝土徐变松弛和桥梁专用温度场分析。

### v1.5.0：发布质量与规模边界

这部分是 v1.5.0 的发布质量目标，也决定项目能否从“功能变多”进入“可持续维护”。

- 建立梁系、平面框架、平面桁架的规模梯度性能基准，记录推荐节点数、构件数、跨数和导出耗时；当前脚本见 [规模与性能基线](verification/performance-baseline.md)。
- 将 Playwright 视觉回归和 DOCX 图形导出矩阵纳入 release 或 nightly workflow。
- 持续把 benchmark 从 61 个推进到 100 个，优先补教材解析解、独立刚度法交叉验证和版本化商业软件对标。
- 保持 `/api/jobs` 为本地轻量异步入口；如要支持生产级队列，应在不破坏现有 API 契约的前提下抽象任务存储和执行边界。

验收口径：

- 性能结论来自可复跑脚本或测试，不写成营销承诺。
- 视觉回归失败能定位到结构类型、视口、页面状态和截图差异。
- 新增公开 benchmark 必须同时提供模型输入、标准值、容许误差和验证来源。
- `release-1-5-load-scenarios.spec.ts` 覆盖梁系和平面桁架工况 / 组合来源切换；DOCX 图形导出仍以三浏览器矩阵为准。

### v1.6.0：外部宿主集成与本地项目契约增强

v1.6.0 的主线不是把 Solver 做成平台，而是在保持免费个人工具边界不变的前提下，让外部系统可以更顺滑地嵌入、调用和保存项目文档。身份体系、远程持久化、组织协作和后台管理能力不进入本仓库。

本版优先任务：

- Host iframe 协议稳定化：定义 launch、ready、project.changed、saveRequest、saveResult 和 error 消息，使用 `sessionId` 与 `nonce` 绑定一次嵌入会话。
- Host origin 边界：前端支持宿主 origin allowlist，默认个人工具模式不受影响。
- 本地项目文件 manifest：`.slv` 当前继续作为单 JSON 项目文件写入，同时声明 `single-json`、`zip-container` 和 `project-folder` 的契约边界，为未来容器格式预留空间。
- Artifact manifest：DOCX / XLSX 导出 metadata 写入 manifest 版本、artifact 类型、项目 manifest、结果来源、诊断摘要和 host 协议版本。
- Integration API 错误码：项目文档无效、不支持的导出格式、不支持的 host 变更、host 保存结果无效、求解失败、导出失败和未知工具都有稳定 `errorCode`。
- 中性模板 registry：公开内置模板、结构类型、可执行动作和 benchmark 映射，不包含身份、组织或远程持久化字段。
- Host iframe demo：提供本地 HTML 示例，演示外部宿主如何加载 Solver、发送项目文档、接收保存请求和回传保存结果。
- JSON Schema registry：公开项目文件 manifest、host message、artifact manifest 和 template registry schema，便于外部系统做契约校验。

验收口径：

- 不登录、不连接远程存储、不引入数据库时，Solver 仍可作为个人本地工具完整使用。
- 外部宿主能通过 demo 完成 launch、项目变更、保存请求和保存结果闭环。
- CLI / integration API 能在无内部 import 的前提下运行项目文档校验、host launch、项目变更、求解、导出 metadata、导出 artifact 和模板 registry。
- 错误分支返回稳定 `errorCode`，外部系统不需要解析中文 warning。
- 项目文件 manifest、host message、artifact manifest 和模板 registry 均有 JSON Schema 和自动化测试覆盖。
- 公开文档只描述中性对接能力，不写入身份、组织、授权或商业化策略。

## 模块重点

| 模块 | 当前重点 | 不应偏离的边界 |
|---|---|---|
| 梁系 | 高频梁型模板、控制截面说明、挠度/弯矩/剪力图、荷载工况 UI 补齐和计算书表达 | 不扩展到复杂桥梁专项、影响线、动力分析或未建模轴向约束温度效应 |
| 二维平面框架 | 门式刚架、显式节点-构件建模、多工况/组合、构件均匀温度荷载、节点位移、杆端内力和构件控制值 | 不引入三维杆系、规范承载力设计、桥梁专用温度场或施工安全专项 |
| 二维平面桁架 | 典型屋架/桥式桁架、节点位移、杆件轴力、轴应力、荷载工况 UI 补齐和稳定性诊断 | 不把桁架杆件当作受弯构件，不输出弯矩主指标，不在理想桁架内引入截面温度梯度 |
| 公共能力 | ASMS-JSON、模板库、公开案例、导出、benchmark、验证等级、错误契约和学习文档 | 不让前端、API、CLI、MCP 各自发明一套模型字段 |

补充约束：桁架计算书不得引入弯矩作为主指标。

## 明确非目标

以下方向不进入当前开源核心，也不作为近中期对外承诺：

- 三维杆系、Frame3D 或空间结构求解。
- 影响线、动力分析、规范级稳定分析、屈曲模态分析、非线性材料、几何非线性和接触问题。
- 规范设计、构件承载力设计、施工安全专项业务和工程签审。
- OpenSeesPy 或其他外部求解器作为运行时内核。
- 热-结构耦合、截面温度梯度、瞬态热传导、混凝土徐变松弛和桥梁专用温度场分析。
- 云协作、多用户平台、组织权限系统和插件化生态。
- 地区规范碎片化适配和商业软件式模块大拼盘。

如果后续需要上述能力，应作为独立项目或独立扩展路线重新立项，不反向牵引当前三类核心模块扩大范围。

## 成功标准

- 外部用户可以按 README 和快速开始文档完成安装、启动、求解和导出。
- 三类核心模块的典型算例可与参考解、教材解或独立刚度法基线稳定对齐。
- 结构工程用户能清楚看到输入、假定、结果、单位、符号约定和适用边界。
- 贡献者可以通过 ASMS-JSON、benchmark 和测试补充新算例。
- 项目对外叙事保持清晰：这是聚焦梁系、二维平面桁架和二维平面框架的现代结构力学求解器，不是小号商业设计平台。

## 维护规则

- 新增路线项必须说明对应结构类型、响应指标、验证方式和非目标影响。
- 新增模板、导出字段或 API 字段必须同步检查 README、能力边界、API 参考和测试。
- 新增会跨越 UI、API、CLI/MCP、文本模型或计算书导出的专业字段时，先更新 `shared/asms-contract-fields.json` 或对应共享词表，再补前端 payload、后端 schema、公开文档和漂移测试。
- 新增计算书图形或结果插图时，先更新 `shared/report-figures.json` 或共享图形目录，再补前端同源 `reportImages`、后端 DOCX 媒体断言和必要的浏览器导出回归。
- 新增内置模板时，必须同步评估 `data/verification/template_benchmark_map.json`；没有直接 benchmark 时应标注“相近/相关”并在路线或任务中记录专项补充需求。
- 中英双语与全球开源定位升级按 [中英双语与全球开源定位方案](global-open-source-i18n-plan.md) 分阶段推进，作为独立长期路线管理。
- 新增功能若不能进入 benchmark、测试或可复核样例，应先作为实验能力处理，不写入主路线承诺。
- 路线图只保留阶段方向和验收口径；具体任务拆分应放在 issue、PRD 或任务清单中。
