# ArchSight 工程文本模型规范

## 目标

工程文本模型用于把梁系、平面框架和平面桁架的计算模型表示为可复制、可审阅、可通过即时通讯传输的纯文本。它服务于三类场景：

- 从微信、邮件、表格或其他结构工具复制参数后快速复核。
- 在讨论中以短文本交换模型，降低截图和口头转述造成的单位误差。
- 将前端参数表、结构示意图和后端求解载荷统一到同一份可追溯输入。

文本框中的内容始终视为草稿。系统可以实时检查语法和工程引用关系，但只有用户点击“应用文本模型”后，才写入正式模型、画布和右侧参数表。

## 通用规则

- 每行一个命令，空行会被忽略。
- `#` 和 `//` 之后的内容视为注释。
- 分隔符支持英文逗号、中文逗号、制表符和空格。
- 命令名大小写不敏感，中文别名仅用于提升可读性。
- 数值单位必须按本规范填写，不在文本中混写单位字符串。
- 存在诊断时不得写入正式模型。诊断包括未知命令、非法数值、引用不存在的节点/构件/支座、被过滤的无效对象等。

## 梁系文本模型

适用于连续梁、简支梁和悬臂梁的跨段、支座、弹簧支座和荷载输入。

### 单位口径

| 参数 | 单位 |
| --- | --- |
| 跨长、支座位置 | m |
| 弹性模量 E | GPa |
| 截面惯性矩 I | cm4 |
| 均布荷载、线性分布荷载 | kN/m |
| 集中力 | kN |
| 集中力位置 | 全梁长度比例 x/L，范围 0-1 |
| 平动弹簧刚度 | kN/m |
| 转动弹簧刚度 | kN·m/rad |

### 命令

```text
SPAN,跨长m,E_GPa,I_cm4
SUPPORT,编号,x位置m,类型
SPRING,支座编号,自由度,刚度
LOAD,uniform,q_kN_per_m
LOAD,linear,q1,q2,startRatio,endRatio
LOAD,point,P_kN,ratio
```

支座类型支持 `fixed`、`pinned`、`roller`、`free`，也支持中文常用表达“固定、固结、滚动、滑动、自由”。弹簧自由度支持 `v` 和 `rz`。

### 示例

```text
# 两跨连续梁，含均布荷载和集中力
SPAN,4,210,4500
SPAN,4,210,4500
SUPPORT,S1,0,pinned
SUPPORT,S2,4,pinned
SUPPORT,S3,8,roller
LOAD,uniform,10
LOAD,point,12,0.75
```

## 平面框架文本模型

适用于显式节点、构件、支座、节点荷载和构件分布荷载。当前文本模型导入基础荷载，不导入高级工况与组合；应用文本模型时会重置既有工况和组合，避免旧组合引用新模型中不存在的对象。

### 单位口径

| 参数 | 单位 |
| --- | --- |
| 节点坐标 | m |
| 弹性模量 E | GPa |
| 截面面积 A | cm2 |
| 截面惯性矩 I | cm4 |
| 节点力 Fx、Fy | kN |
| 节点弯矩 Mz | kN·m |
| 构件线荷载 | kN/m |

### 命令

```text
N,节点号,x,y
NODE,节点号,x,y,支座类型
NSUPT,节点号,支座码,角度deg
E,起点节点号,终点节点号,...释放参数
MEMBER,构件号,起点节点号,终点节点号,类型,E_GPa,A_cm2,I_cm4
PROP,构件序号或范围,E_GPa,A_cm2,I_cm4,类型
NLOAD,节点号,方向码,荷载值
LOAD,节点号,Fx_kN,Fy_kN,Mz_kN_m
DLOAD,构件序号,qStart,qEnd,方向
ELOAD,构件序号,荷载类型,荷载值,方向
```

`N`、`E`、`NSUPT`、`PROP`、`NLOAD`、`DLOAD` 兼容常见结构程序的短命令风格。`MEMBER` 和 `LOAD` 提供更直观的 ArchSight 显式写法。

### 示例

```text
# 单跨门式框架
N,1,0,0
N,2,6,0
N,3,0,4
N,4,6,4
NSUPT,1,6,0
NSUPT,2,6,0
E,1,3,1,1,1,1,1,1
E,3,4,1,1,1,1,1,1
E,2,4,1,1,1,1,1,1
PROP,2,2,210,220,15000,beam
DLOAD,2,-18,-18,global_y
NLOAD,4,1,24,0
```

## 平面桁架文本模型

适用于二维铰接杆系。桁架文本模型只导入节点、杆件和节点荷载，主指标为节点位移、杆件轴力、杆件轴应力和支座反力，不导入构件弯矩作为主指标。

### 单位口径

| 参数 | 单位 |
| --- | --- |
| 节点坐标 | m |
| 弹性模量 E | GPa |
| 截面面积 A | cm2 |
| 节点力 Fx、Fy | kN |

### 命令

```text
NODE,节点号,x,y,支座类型
MEMBER,杆件号,起点节点号,终点节点号,E_GPa,A_cm2,类型
LOAD,节点号,Fx_kN,Fy_kN
```

支座类型支持 `pinned`、`roller`、`free`，也支持中文常用表达“铰接、铰支、滚动、滑动”。

### 示例

```text
# 三角桁架
NODE,N1,0,0,pinned
NODE,N2,6,0,roller
NODE,N3,3,3,free
MEMBER,M1,N1,N3,210,24,upper_chord
MEMBER,M2,N3,N2,210,24,upper_chord
MEMBER,M3,N1,N2,210,24,bottom_chord
LOAD,N3,0,-50
```

## 检查结果面板

文本输入区下方的检查结果面板必须至少表达：

- 是否可应用。
- 将导入或更新的对象数量，例如节点、构件、杆件、跨段、支座、荷载。
- 单位口径，防止将 N、mm、MPa 等单位误填为 kN、m、GPa。
- 诊断信息及被忽略对象。存在诊断时，“应用文本模型”不得写入正式模型。

## 后续扩展

- 增加粘贴适配器：Excel/CSV、结构软件短命令、Markdown 表格可先转换为内部规范文本，再进入统一检查流程。
- 增加差异预览：应用前显示“当前模型”和“文本模型”的对象数量与关键参数差异。
- 增加实时预览开关：默认关闭，仅当用户显式启用且文本无诊断时更新临时预览，不直接覆盖正式模型。
