# 核心功能导览与操作说明

适用版本：v1.1.0
适用对象：结构工程师、土木/结构方向教师、高校学生、结构工程工具开发者

ArchSight Solver 是一个面向结构工程工作台的开源结构力学求解器，当前核心能力不是“单个梁计算器”，而是把三类常见线弹性静力分析、模型复用、结果复核和计算书导出串成一条完整工作流。

## 1. 当前能解决什么问题

- 梁系分析：用于连续梁、简支梁、悬臂梁等常见梁系工况，重点复核挠度、弯矩、剪力和支座反力。
- 平面框架分析：用于门式刚架、二维框架和梁柱体系，重点复核节点位移、构件弯矩、杆端内力和支座反力。
- 平面桁架分析：用于典型屋架、支撑体系和教学桁架算例，重点复核节点位移、杆件轴力、轴应力和支座反力。
- 敏感性分析：用于判断跨长、荷载、弹性模量、截面参数等扰动对结果指标的影响。
- 模板与文本模型：用于复用典型工况、导入导出结构化文本模型，减少重复录入。
- 计算书导出：用于生成 Word / Excel 结果文件，便于复核、教学和阶段性交付。
- API / CLI / MCP 集成：用于自动化计算、Agent 调用、JSON Schema 契约发现和 benchmark 验证。

## 2. 推荐工作流

1. 新建或打开项目文件。
2. 在工程树中新建或选择分析对象：连续梁、平面框架或平面桁架。
3. 在右侧参数面板选择模板，或在“文本”页导入已有模型。
4. 在“基本 / 对象 / 表格”中核对材料、几何、支座、荷载和截面参数。
5. 在中部“参数建模”页检查结构预览，确认荷载方向和支座约束正确。
6. 切换到“结构计算”，运行计算并先看“全部结果”。
7. 对异常指标切换到位移、弯矩、剪力、轴力或支座反力等单项结果复核。
8. 必要时运行敏感性分析，判断控制参数对结果的影响。
9. 保存模板，或导出 Word / Excel 计算书。

## 3. 右侧参数面板怎么用

三个分析模块统一使用五类入口：

- 模板：套用内置典型工况，适合快速开始和教学演示。
- 基本：查看模型口径、单位、主要结果指标和校核方法简介。
- 对象：编辑节点、构件/跨段、支座、荷载以及当前选中对象属性。
- 文本：导入或导出完整文本模型，适合批量维护和版本对比。
- 表格：查看或编辑更完整的节点、构件、支座、荷载参数表。

建议先从“模板”进入，再用“对象”和“表格”精修参数；复杂模型优先用“文本”导入，减少界面逐项录入。

## 4. 三类结构如何选择

### 梁系

适合：连续梁、简支梁、悬臂梁、多跨梁、局部均布荷载、集中荷载和线性分布荷载。

重点检查：

- 跨段长度、弹性模量、截面惯性矩。
- 支座类型和荷载方向。
- 最大挠度、弯矩图、剪力图、支座反力。

### 平面框架

适合：门式刚架、二维梁柱框架、带节点荷载和构件荷载的平面杆系。

重点检查：

- 节点坐标、构件连接、截面面积、截面惯性矩。
- 固定、铰支、滚动等支座约束。
- 节点水平/竖向位移、构件弯矩、杆端轴力/剪力/弯矩、支座反力。

### 平面桁架

适合：铰接杆系、屋架、支撑体系、只承受节点荷载的桁架算例。

重点检查：

- 节点坐标、杆件连接、截面面积、材料参数。
- 节点荷载方向和支座约束。
- 节点位移、杆件轴力、轴应力、支座反力。

桁架不以弯矩作为主结果；如果需要弯矩、剪力和转角，应使用平面框架模块。

## 5. 结果如何复核

推荐复核顺序：

1. 先看结构预览：几何、支座、荷载方向是否正确。
2. 再看结果摘要：控制值、控制位置和计算结论是否合理。
3. 再看位移结果：变形方向和最大值量级是否符合结构力学直觉。
4. 再看内力结果：梁看弯矩/剪力，框架看构件内力，桁架看杆件轴力/轴应力。
5. 最后看支座反力：方向、量级和整体平衡是否异常。

常见单位：

- 长度：m
- 位移 / 挠度：mm
- 力：kN
- 分布荷载：kN/m
- 弯矩：kN·m
- 弹性模量：GPa
- 截面惯性矩：cm^4

## 6. 敏感性分析怎么用

敏感性分析用于回答“哪个参数变化会明显影响结果”。

推荐用法：

1. 先完成一次正常结构计算。
2. 选择扰动参数，例如跨长、荷载、弹性模量、截面惯性矩或截面面积。
3. 选择响应指标，例如最大挠度、最大弯矩、最大节点位移或最大杆件轴力。
4. 设置扰动范围和步数。
5. 查看响应曲线，判断结果是否对某个参数高度敏感。

注意：敏感性分析用于趋势判断和方案复核，不替代完整结构验算。

## 7. 模板、文本模型和项目文件

- 模板库：保存教材算例、标准复核工况、常用截面工况；后续可一键恢复。
- 文本模型：适合复制、审阅、版本对比和批量生成结构模型。
- 项目文件：保存当前工作台状态、分析对象和结果上下文，便于继续编辑。

如果只是快速试算，用模板最快；如果要长期维护模型，用文本模型和项目文件更稳。

## 8. 导出与交付

计算完成后可导出：

- Word 计算书：适合阅读、教学讲解、方案阶段复核。
- Excel 数据表：适合核对节点、构件、支座、荷载和结果数据。

导出前建议确认：

- 结构预览无明显建模错误。
- 结果摘要中的控制指标量级合理。
- 敏感性分析或异常结果已复核。
- 项目信息和分析对象名称清晰。

## 9. API、CLI、MCP 与 benchmark

v1.1.0 面向自动化和 Agent 集成补齐了工程接口：

- `/api/jobs`：提交异步求解作业，适合批量计算和长任务。
- `/api/contracts/schemas`：获取 JSON Schema 契约，便于前端、CLI、MCP 和 Agent 共用同一输入输出规范。
- `archsight-solver-tool`：通用 CLI 入口，可执行求解、敏感性分析和 benchmark。
- MCP 工具：面向 Agent 暴露通用计算、敏感性分析、benchmark 列表与运行能力。
- benchmark catalog / runner / report：用于公开验证典型算例和 CI 回归。

普通工程用户可以主要使用 Web 工作台；开发者和 Agent 工作流可以使用 API、CLI 和 MCP。

## 10. 常见判断

### 计算失败

优先检查节点、构件、支座、荷载是否完整，数值是否为正，框架或桁架是否存在机构。

### 结果量级明显不合理

优先检查单位、弹性模量、截面惯性矩、截面面积、荷载方向和支座约束。

### 不知道该选框架还是桁架

如果杆件只按轴力工作且节点近似铰接，选平面桁架；如果需要考虑弯矩、剪力、刚接节点或构件转角，选平面框架。

## 11. 使用边界

本工具用于计算辅助、教学演示、方案阶段复核和工程软件原型验证，不替代规范设计软件和注册工程师审查。导出的计算书和自动化接口结果都应进行人工复核。
